# 🔐 Clerk Docs Markdown

> **The entire Clerk documentation, optimized for LLMs.**

An offline-first, AI-ready collection of the complete [Clerk Documentation](https://clerk.com/docs). Every quickstart, guide, SDK reference, and API endpoint — converted from Clerk's MDX source into clean, structured Markdown.

---

## ✨ Features

- 🏎️ **Optimized for LLMs**: MDX components (Tabs, Accordions, Properties) converted into standard markdown for perfect RAG and context window usage.
- 📂 **Structured Layout**: Mirrors the official docs with `getting-started/`, `guides/`, and `reference/`.
- 💻 **Multi-Framework**: Contains all framework specific guides (Next.js, React, Astro, Vue, Expo, Nuxt, Remix, React Router, etc.).
- 🔍 **Inlined Partials**: Shared snippets (`<Include>`) are fully resolved recursively.
- 📄 **Clean Markdown**: Zero JSX fluff, just pure documentation context.

## 📁 Repository Structure

```text
docs/
├── getting-started/      # Quickstarts for every framework
├── guides/               # Core concepts, auth flows, organizations, etc.
└── reference/            # SDKs, Hooks, Components, Backend API, Frontend API
```

See [structure.md](structure.md) for a full hierarchical list of all 700+ documents.

## 🚀 Getting Started

If you're building a project using an AI assistant (like Claude, ChatGPT, or Cursor) and need deep Clerk context, simply point your AI to this repository.

### Example Uses
- Ask standard questions: *"Read `docs/getting-started/quickstart.nextjs.md` and set up my Next.js App Router."*
- Troubleshooting: *"Check `docs/guides/development/errors` to figure out why I'm getting a token error."*
- Deep dives: *"Scan all of `docs/reference/backend` to understand how the JS Backend API manages user metadata."*

## 🛠️ How it was built

This mirror is automatically generated by cloning the open-source [clerk/clerk-docs repo](https://github.com/clerk/clerk-docs) and running a custom Python MDX-to-Markdown converter (`convert_docs.py`) that strips React components, resolves partials, and normalizes code syntax definitions for AI parsing.

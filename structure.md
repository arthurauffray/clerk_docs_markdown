# Clerk Documentation Structure

This document describes the structure of the Clerk documentation.

## Hierarchy

The documentation mirrors the [Clerk docs](https://clerk.com/docs) hierarchy:

1. **Getting Started**: Quickstarts for every framework/SDK
2. **Guides**: Authentication flows, configuration, security, billing, organizations, development, and more
3. **Reference**: SDK references (Next.js, React, Vue, Astro, Expo, Express, Go, Ruby, etc.), components, hooks, Backend API, and JavaScript API
